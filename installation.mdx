## Step 1: Install Flutter SDK

<Steps>

### Download Flutter

Visit the official Flutter website:

<CodeGroup>

```bash macOS
# Download Flutter SDK
curl -O https://storage.googleapis.com/flutter_infra_release/releases/stable/macos/flutter_macos_3.7.2-stable.zip

# Extract the archive
unzip flutter_macos_3.7.2-stable.zip

# Move to desired location
sudo mv flutter /usr/local/flutter
```

```bash Windows (PowerShell)
# Download from:
# https://storage.googleapis.com/flutter_infra_release/releases/stable/windows/flutter_windows_3.7.2-stable.zip

# Extract to C:\src\flutter
# Add to PATH via System Environment Variables
```

```bash Linux
# Download Flutter SDK
wget https://storage.googleapis.com/flutter_infra_release/releases/stable/linux/flutter_linux_3.7.2-stable.tar.xz

# Extract the archive
tar xf flutter_linux_3.7.2-stable.tar.xz

# Move to desired location
sudo mv flutter /usr/local/flutter
```

</CodeGroup>

### Add Flutter to PATH

Make Flutter accessible from your terminal:

<CodeGroup>

```bash macOS/Linux (Bash)
# Edit your shell profile
nano ~/.bashrc  # or ~/.zshrc for zsh

# Add this line:
export PATH="$PATH:/usr/local/flutter/bin"

# Apply changes
source ~/.bashrc  # or source ~/.zshrc
```

```bash macOS/Linux (Fish)
# Edit Fish config
nano ~/.config/fish/config.fish

# Add this line:
set -gx PATH $PATH /usr/local/flutter/bin

# Reload config
source ~/.config/fish/config.fish
```

```powershell Windows (PowerShell as Admin)
# Add to System PATH
[System.Environment]::SetEnvironmentVariable(
  'Path',
  [System.Environment]::GetEnvironmentVariable('Path', 'Machine') + ';C:\src\flutter\bin',
  'Machine'
)
```

</CodeGroup>

### Verify Installation

Confirm Flutter is correctly installed:

```bash
flutter --version
```

Expected output:

```
Flutter 3.7.2 • channel stable
Framework • revision xxx
Engine • revision xxx
Tools • Dart 3.7.2 • DevTools 2.x.x
```

### Run Flutter Doctor

Check for any missing dependencies:

```bash
flutter doctor -v
```

This command checks:
- ✓ Flutter SDK installation
- ✓ Connected devices
- ✓ Android toolchain (if developing for Android)
- ✓ Xcode (if developing for iOS on macOS)
- ✓ VS Code / Android Studio setup

<Warning>
  Address any issues marked with [✗] or [!] before proceeding.
</Warning>

</Steps>

## Step 2: Set Up Development Environment

Choose your preferred IDE:

<Tabs>

      "dart.flutterSdkPath": "/usr/local/flutter",

    1. Open Android Studio

## Step 5: Firebase Setup

<Steps>

### Install Firebase CLI

Install globally via npm:

```bash
npm install -g firebase-tools
```

Verify installation:

```bash
firebase --version
```

### Install FlutterFire CLI

```bash
dart pub global activate flutterfire_cli
```

Add to PATH if needed:

```bash
# macOS/Linux
export PATH="$PATH:$HOME/.pub-cache/bin"

# Add to ~/.bashrc or ~/.zshrc for persistence
```

### Authenticate with Firebase

```bash
firebase login
```

This opens a browser for Google authentication.

### Create Firebase Project

<Tabs>
  <Tab title="Via Console">
    1. Go to [Firebase Console](https://console.firebase.google.com/)
    2. Click **Add project**
    3. Enter project name (e.g., "restaurant-reservations")
    4. Accept terms and click **Continue**
    5. Disable Google Analytics (or configure)
    6. Click **Create project**
    7. Wait for provisioning to complete
  </Tab>
  
  <Tab title="Via CLI">
    ```bash
    # Create new Firebase project
    firebase projects:create restaurant-reservations
    
    # Set default project
    firebase use restaurant-reservations
    ```
  </Tab>
</Tabs>

### Enable Firestore Database

1. In Firebase Console, navigate to **Build** → **Firestore Database**
2. Click **Create database**
3. Select location (choose closest region)
4. Start in **Test mode** for development
5. Click **Enable**

<Warning>
  Test mode allows unrestricted access. Update security rules before production:
  
  ```javascript
  rules_version = '2';
  service cloud.firestore {
    match /databases/{database}/documents {
      match /{document=**} {
        allow read, write: if request.auth != null;
      }
    }
  }
  ```
</Warning>

### Enable Firebase Authentication

1. Go to **Build** → **Authentication**
2. Click **Get started**
3. Enable providers:
   - **Email/Password**: Toggle on
   - **Google**: Toggle on, add support email
4. Click **Save**

### Configure Firebase Hosting (Optional)

For web deployment:

```bash
# Initialize hosting
firebase init hosting

# Select options:
# - Public directory: build/web
# - Single-page app: Yes
# - GitHub auto-deploy: No (or Yes if desired)
```

</Steps>

## Step 6: Clone and Configure Project

<Steps>

### Clone Repository

```bash
git clone <repository-url>
cd app_restaurante
```

### Install Dependencies

```bash
flutter pub get
```

This installs packages from `pubspec.yaml`:

```yaml pubspec.yaml
dependencies:
  flutter:
    sdk: flutter
  
  # Firebase
  firebase_core: ^3.3.0
  firebase_auth: ^5.1.0
  cloud_firestore: ^5.6.12
  google_sign_in: ^6.2.1
  
  # State Management
  flutter_bloc: ^8.1.3
  
  # Dependency Injection
  get_it: ^7.6.4
  
  # Routing
  go_router: ^13.0.0
  
  # Utilities
  uuid: ^4.0.0
```

### Configure Firebase for Flutter

Run FlutterFire configuration:

```bash
flutterfire configure
```

**Configuration steps:**

1. **Select Firebase project**: Choose your created project
2. **Select platforms**: 
   - [x] Android
   - [x] iOS
   - [x] Web
   - [ ] macOS (optional)
   - [ ] Windows (optional)
3. **Android package name**: Use `com.example.app_restaurante` or customize
4. **iOS bundle ID**: Use `com.example.appRestaurante` or customize

<Info>
  This generates `lib/firebase_options.dart` with platform-specific configuration.
</Info>

### Create Firebase Configuration File

Create `.firebaserc` in project root:

```json .firebaserc
{
  "projects": {
    "default": "your-firebase-project-id"
  }
}
```

<Note>
  Replace `your-firebase-project-id` with your actual project ID from Firebase Console.
</Note>

### Configure Firestore Security Rules

Update `firestore.rules`:

```javascript firestore.rules
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Business profiles - owners only
    match /negocios/{negocioId} {
      allow read: if true;
      allow write: if request.auth != null && request.auth.uid == resource.data.propietarioId;
    }
    
    // Tables - public read, owner write
    match /mesas/{mesaId} {
      allow read: if true;
      allow write: if request.auth != null;
    }
    
    // Reservations - authenticated users
    match /reservas/{reservaId} {
      allow read: if request.auth != null;
      allow create: if true; // Allow guest reservations
      allow update, delete: if request.auth != null;
    }
    
    // Operating hours - public read, owner write
    match /horarios/{horarioId} {
      allow read: if true;
      allow write: if request.auth != null;
    }
  }
}
```

Deploy rules:

```bash
firebase deploy --only firestore:rules
```

### Download Google Services Files

<Tabs>
  <Tab title="Android">
    1. In Firebase Console, go to **Project Settings**
    2. Select your Android app
    3. Download `google-services.json`
    4. Place in: `android/app/google-services.json`
  </Tab>
  
  <Tab title="iOS">
    1. In Firebase Console, go to **Project Settings**
    2. Select your iOS app
    3. Download `GoogleService-Info.plist`
    4. Place in: `ios/Runner/GoogleService-Info.plist`
    
    **Update Xcode**:
    - Open `ios/Runner.xcworkspace` in Xcode
    - Drag `GoogleService-Info.plist` into Runner folder
    - Ensure "Copy items if needed" is checked
  </Tab>
</Tabs>

</Steps>

## Step 7: Initialize Database

Set up initial data in Firestore:

<Tabs>
  <Tab title="Via Firebase Console">
    ### Create Business Collection
    
    1. Go to Firestore Database
    2. Click **Start collection**
    3. Collection ID: `negocios`
    4. Add first document:
    
    ```json
    {
      "nombre": "Demo Restaurant",
      "email": "owner@demo.com",
      "telefono": "+1234567890",
      "direccion": "123 Main Street",
      "propietarioId": "<your-auth-uid>",
      "maxDiasAnticipacionReserva": 14,
      "duracionPromedioMinutos": 60,
      "imagenUrl": null
    }
    ```
    
    ### Create Tables Collection
    
    Collection ID: `mesas`
    
    ```json
    // Mesa 1
    {
      "nombre": "Mesa 1",
      "capacidad": 2,
      "negocioId": "<negocio-document-id>",
      "activa": true
    }
    
    // Mesa 2
    {
      "nombre": "Mesa 2",
      "capacidad": 4,
      "negocioId": "<negocio-document-id>",
      "activa": true
    }
    ```
    
    ### Create Operating Hours
    
    Collection ID: `horarios`
    
    ```json
    {
      "negocioId": "<negocio-document-id>",
      "diaSemana": 1,  // Monday = 1, Sunday = 7
      "horaApertura": "11:00",
      "horaCierre": "22:00",
      "cerrado": false
    }
    ```
    
    Repeat for all 7 days of the week.
  </Tab>
  
  <Tab title="Via Import Script">
    Create a seed data file:
    
    ```json seed_data.json
    {
      "negocios": [
        {
          "nombre": "Demo Restaurant",
          "email": "owner@demo.com",
          "telefono": "+1234567890",
          "direccion": "123 Main Street",
          "maxDiasAnticipacionReserva": 14,
          "duracionPromedioMinutos": 60
        }
      ],
      "mesas": [
        { "nombre": "Mesa 1", "capacidad": 2 },
        { "nombre": "Mesa 2", "capacidad": 4 },
        { "nombre": "Mesa 3", "capacidad": 6 }
      ]
    }
    ```
    
    Import using Firebase CLI or a custom Dart script.
  </Tab>
</Tabs>

## Troubleshooting Common Issues

<AccordionGroup>
  <Accordion title="Flutter doctor shows errors">
    **Android toolchain issues:**
    ```bash
    flutter doctor --android-licenses
    ```
    
    **Xcode issues (macOS):**
    ```bash
    sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer
    sudo xcodebuild -runFirstLaunch
    ```
    
    **Command-line tools not found:**
    - Open Android Studio → SDK Manager → SDK Tools
    - Install "Android SDK Command-line Tools"
  </Accordion>
  
  <Accordion title="Firebase initialization fails">
    **Error**: "FirebaseException: [core/no-app]"
    
    **Solution**:
    1. Verify `firebase_options.dart` exists
    2. Check `main.dart` initializes Firebase:
    ```dart
    await Firebase.initializeApp(
      options: DefaultFirebaseOptions.currentPlatform,
    );
    ```
    3. Run `flutterfire configure` again
  </Accordion>
  
  <Accordion title="Google services files not found">
    **Android**: Place `google-services.json` in `android/app/`
    
    **iOS**: 
    1. Open `ios/Runner.xcworkspace` in Xcode
    2. Right-click Runner folder
    3. Add Files to "Runner"
    4. Select `GoogleService-Info.plist`
    5. Check "Copy items if needed"
  </Accordion>
  
  <Accordion title="Gradle build errors (Android)">
    **Error**: "Could not resolve firebase-bom"
    
    **Solution**: Update `android/build.gradle.kts`:
    ```kotlin
    dependencies {
      classpath("com.google.gms:google-services:4.4.0")
    }
    ```
    
    Then run:
    ```bash
    cd android
    ./gradlew clean
    cd ..
    flutter clean
    flutter pub get
    flutter run
    ```
  </Accordion>
  
  <Accordion title="Pod install fails (iOS)">
    **Solution**:
    ```bash
    cd ios
    pod repo update
    pod install --repo-update
    cd ..
    flutter clean
    flutter run
    ```
    
    If still failing:
    ```bash
    sudo gem install cocoapods
    pod setup
    ```
  </Accordion>
</AccordionGroup>
